Version callback pattern (04-RESEARCH.md Pattern 2, lines 200–227 — adapt verbatim):
```python
def _version_cb(value: bool) -> None:
    if value:
        import importlib.metadata
        try:
            v = importlib.metadata.version("logpose-launcher")
        except importlib.metadata.PackageNotFoundError:
            v = "unknown"
        typer.echo(f"logpose {v}")
        raise typer.Exit()

@app.callback()
def _root(
    version: bool = typer.Option(
        None, "--version",
        callback=_version_cb, is_eager=True,
        help="Show version and exit.",
    ),
) -> None:
    """logpose — multi-game dedicated server launcher."""
```

Key points (enforced by 04-RESEARCH Pitfall 4 + Pitfall 6):
- `raise typer.Exit()` (no code) after echoing version — intent is "clean quit", NOT error.
- Wrap `importlib.metadata.version` in try/except for uninstalled checkouts — fall back to `"unknown"`.
- `is_eager=True` so `--version` short-circuits subcommand parsing.
- The callback docstring (`"""logpose — multi-game dedicated server launcher."""`) surfaces as `logpose --help`'s top-level description per Typer.

Root Typer must become:
```python
app = typer.Typer(
    help="logpose — multi-game dedicated server launcher.",
    no_args_is_help=True,
)
```
