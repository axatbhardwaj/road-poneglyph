1. Update `road_poneglyph/templates/satisfactory.service.template` to include an ExecStop directive. The ExecStop must:
   - Call the Satisfactory HTTPS API SaveGame endpoint
   - Use the cached token file
   - Handle failures gracefully (if API is unreachable — e.g., server crashed — stop should still proceed)
   - Use a bash one-liner with curl (available on all target systems; avoids needing Python in ExecStop path)

Add the ExecStop line BEFORE the ExecStart line (systemd convention):

```ini
[Service]
Environment=SteamAppId=1690800
{auto_update_line}ExecStop=/bin/bash -c 'TOKEN=$(cat {token_path} 2>/dev/null) && curl -sk -X POST https://localhost:{port}/api/v1 -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" -d \'{{"function":"SaveGame","data":{{"SaveName":"pre-shutdown"}}}}\' --max-time 15 || true; sleep 2; kill -INT $MAINPID'
ExecStart={exec_start_path} -Port={port} -ReliablePort={reliable_port} -multihome=0.0.0.0 -log -unattended
```

Wait — systemd ExecStop semantics: When ExecStop is defined, systemd runs it INSTEAD of sending KillSignal directly. The ExecStop script must itself send the signal. The `$MAINPID` variable is available in ExecStop. After ExecStop completes (or TimeoutStopSec expires), systemd sends SIGKILL to remaining processes.

So the template becomes:
```ini
ExecStop=/bin/bash -c 'TOKEN=$(cat {token_path} 2>/dev/null) && curl -sk -X POST https://localhost:{port}/api/v1 -H "Content-Type: application/json" -H "Authorization: Bearer $$TOKEN" -d \'{{"function":"SaveGame","data":{{"SaveName":"pre-shutdown"}}}}\' --max-time 15 || true; sleep 2; kill -INT $$MAINPID || true'
```

NOTE on escaping: In a systemd unit file, `$` for shell variables must be `$$` to prevent systemd from interpreting them. But since we use Python `.format()` with `{token_path}` etc., and the file is a Python format template, we need:
- `{{` and `}}` to escape literal braces in JSON (already needed for .format())
- `$$` is NOT needed because systemd reads the final rendered file (no double-processing). Actually wait — the rendered output IS the systemd file. In systemd unit files, `$MAINPID` is a systemd special variable that systemd resolves, NOT a shell variable. So we want literal `$MAINPID` in the output. For Python .format(), `$` has no special meaning, so it passes through unchanged.
- For the shell variable `$TOKEN`, since this runs in bash via ExecStop, shell interprets `$TOKEN` fine.
- The `{` and `}` in JSON must be `{{` and `}}` in the Python format template.

Final template line (correct escaping for Python .format() + valid bash + valid JSON):

```
ExecStop=/bin/bash -c 'TOKEN=$(cat {token_path} 2>/dev/null) && curl -sk -X POST https://localhost:{port}/api/v1 -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" -d \'{{"function":"SaveGame","data":{{"SaveName":"pre-shutdown"}}}}\' --max-time 15 || true; sleep 2; kill -INT $MAINPID || true'
```

Hmm, but single quotes in bash can't be nested. The curl -d argument uses single quotes inside an already-single-quoted ExecStop. Use double quotes for the -d argument instead:

```
ExecStop=/bin/bash -c 'TOKEN=$(cat {token_path} 2>/dev/null) && curl -sk -X POST https://localhost:{port}/api/v1 -H "Content-Type: application/json" -H "Authorization: Bearer $TOKEN" -d "{{\\"function\\":\\"SaveGame\\",\\"data\\":{{\\"SaveName\\":\\"pre-shutdown\\"}}}}" --max-time 15 || true; sleep 2; kill -INT $MAINPID || true'
```

Actually this is getting complex. Use a simpler approach — put the JSON in a variable:

Better approach: Use `/bin/sh -c` with heredoc-style or just accept that if token is missing, we skip save. Simplest reliable approach:

```

```

Actually, the cleanest approach is to use a separate small script or to use Python as the ExecStop command (since it's already installed):


But that requires the module to be installed in the service user's Python path. The service runs as `{user}` so it may not have road-poneglyph installed.

2. Update `_render_satisfactory_service` in `main.py` to pass the new `{token_path}` placeholder:

```python
def _render_satisfactory_service(
    *,
    user: str,
    working_directory: Path,
    exec_start_path: Path,
    port: int,
    reliable_port: int,
    auto_update: bool,
) -> str:
    """Render satisfactory.service.template with Satisfactory-specific placeholders."""
    template = _get_template("satisfactory.service.template")
    if auto_update:
        auto_update_line = (
            f"ExecStartPre=/usr/games/steamcmd +force_install_dir "
            f'"{working_directory}" +login anonymous +app_update 1690800 validate +quit\n'
        )
    else:
        auto_update_line = ""
    token_path = Path(f"/home/{user}/.config/road-poneglyph/satisfactory-api-token")
    return template.format(
        user=user,
        port=port,
        reliable_port=reliable_port,
        exec_start_path=exec_start_path,
        working_directory=working_directory,
        auto_update_line=auto_update_line,
        token_path=token_path,
    )
```

Note: Use `/home/{user}/...` not `~` because systemd does not expand `~` in ExecStop.
