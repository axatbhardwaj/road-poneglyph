  <action>
On the VM (all commands assume non-root user with sudo access; `logpose` is on PATH from
the venv):

1. **Pre-install snapshot:**
   ```
   date --iso=seconds
   cat /etc/os-release
   getent passwd steam || echo "steam user absent (expected)"
   test -f /usr/local/bin/arkmanager && echo "arkmanager present" || echo "arkmanager absent (expected)"
   test -f /etc/sudoers.d/logpose-ark && echo "sudoers fragment present" || echo "sudoers fragment absent (expected)"
   ss -tuln | grep -E ':(7777|7778|27015|27020)\b' || echo "ARK ports free (expected)"
   df -h /home
   free -g
   ```

2. **Run install** (replicating install record §4 with logpose):
   ```
   logpose ark install \
     --map TheIsland \
     --session-name "e2e-debian13-ark" \
     --admin-password 33-22-11-00 \
     --port 7778 \
     --query-port 27015 \
     --rcon-port 27020 \
     --players 10 \
     --beta preaquatica \
     --enable-autostart \
     --start \
     2>&1 | tee /tmp/logpose-ark-install.log
   ```

   Capture the full log. Expect runtime ~5–15 minutes (18 GB download).

3. **Post-install verification (run each, record output):**
   ```
   # ARK-16: steam user created
   getent passwd steam

   # ARK-14: arkmanager installed
   ls -la /usr/local/bin/arkmanager
   arkmanager --version || /usr/local/bin/arkmanager --version

   # ARK-10: main.cfg seeded with 10 keys
   sudo cat /etc/arkmanager/instances/main.cfg | grep -E '^(arkserverroot|serverMap|ark_)' | sort

   # ARK-13: RCON triad aligned
   sudo grep -E 'ark_RCON(Enabled|Port)|ark_ServerAdminPassword' /etc/arkmanager/instances/main.cfg

   # Pitfall 5: main.cfg permissions tightened
   sudo stat -c '%a %U:%G' /etc/arkmanager/instances/main.cfg   # expect 640 root:steam

   # ARK-18: sudoers fragment installed and valid
   sudo ls -la /etc/sudoers.d/logpose-ark    # expect -r--r----- 1 root root
   sudo cat /etc/sudoers.d/logpose-ark
   sudo visudo -c                            # expect exit 0

   # ARK-17: install actually downloaded files (double-call quirk was worked around)
   sudo ls -la /home/steam/ARK/ShooterGame/Binaries/Linux/ShooterGameServer

   # ARK-19: verb delegation works — status via arkmanager (not systemctl)
   sudo -u steam /usr/local/bin/arkmanager status

   # POL-01 + POL-03 + POL-05: merged polkit rule covers both units
   sudo cat /etc/polkit-1/rules.d/40-logpose.rules | head -20   # expect palserver AND arkserver in units array

   # POL-05 (ARK): if --enable-autostart was used, pkcheck against arkserver.service
   pkcheck --action-id=org.freedesktop.systemd1.manage-units \
           --process $$ \
           --detail unit arkserver.service \
           && echo "POL-05 ARK: allowed"
   ```

4. **Behavior verification:**
   ```
   # ARK-07 port probe worked (no collision since ports were free pre-install)
   # ARK-06 map validation worked (install didn't fail with typer.BadParameter)
   # Launch args resolved correctly (from ps):
   ps auxf | grep ShooterGameServer | grep -v grep

   # Wait up to 10 min for first map load, then check listening:
   sleep 600 || true
   ss -tuln | grep -E ':(7778|27015|27020)\b'
   sudo -u steam /usr/local/bin/arkmanager status    # expect Server listening: Yes
   ```

5. **logpose verb smoke (warm control path via sudoers fragment):**
   ```
   logpose ark status    # must succeed without sudo prompt (sudoers NOPASSWD)
   logpose ark stop      # graceful save
   sleep 30
   sudo -u steam /usr/local/bin/arkmanager status   # Server running: No
   logpose ark start
   sleep 60
   logpose ark status    # Server running: Yes
   ```

6. **edit-settings round-trip:**
   Interactive — edit `ark_MaxPlayers` from 10 → 20, save, inspect:

   # Expect ark_MaxPlayers="20" and all other keys intact (ARK-09 round-trip preservation)
   ```

7. **Idempotency check:**

   #         "arkmanager already installed; skipping netinstall.",
   #         apt fast (already-installed packages),
   #         main.cfg re-seeded to new values.
   ```
